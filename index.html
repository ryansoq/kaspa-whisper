How It Works

Alice sends a message to Bob by locking 0.2 tKAS into a covenant P2SH script with a CLTV timeout. When Bob reads the message (spends the UTXO), the covenant enforces that 0.2 tKAS is refunded to Alice. If Bob doesn't read it, Alice can reclaim after the timeout. Net cost: only mining fees (~0.0001 tKAS).

Alice (sender) Covenant UTXO Bob (receiver) │ │ │ ├── lock 0.2 KAS ────────►│ P2SH script │ │ + encrypted payload │ IF: refund → A (Bob) │ │ + CLTV timeout │ ELSE: reclaim (Alice) │ │ │ │ │ Path 1: Bob reads │◄── Bob spends (reads) ───┤ │◄── 0.2 KAS refund ──────│ covenant verifies: │ │ (enforced by script) │ ✓ output[0] → Alice │ │ │ ✓ amount ≥ 0.2 KAS │ │ │ ✓ signed by Bob │ │ │ │ │ Path 2: Timeout │ │ ├── reclaim 0.2 KAS ◄─────│ CLTV verifies: │ │ (after timeout) │ ✓ DAA ≥ timeout │ │ │ ✓ signed by Alice │

🛡️Security Model

🏠 Local (Safe Zone)
  • Private keys
  • ECIES encryption / decryption
  • Transaction signing
  • Message plaintext
🌐 Network (Public Zone)
  • Pre-signed transactions
  • Encrypted ciphertext only
  • Covenant info (public metadata)
  • P2SH addresses
🔐

ECIES Encryption

Messages encrypted with recipient's public key (secp256k1). Only the recipient's private key can decrypt. Same keypair as Kaspa wallet.

🤝

Trustless Refund

Covenant script enforces: recipient can only spend by refunding sender. No intermediary, no trust needed.

📍

SPK Pinning

OP_TX_OUTPUT_SPK ensures refund goes to exact sender address. Can't be redirected.

✍️

Schnorr Signatures

Only designated recipient's signature can unlock the covenant UTXO. 32-byte x-only pubkeys.

🔌Offline Decode (No API Needed)

🛡️ Fully Self-Contained

If the API is down, you can still decrypt and refund using the TX payload alone. All covenant metadata lives in the a field on-chain.

Option 1: Pass payload directly

# Get the TX payload from a block explorer or kaspad, then:
python3 decode.py \
  --tx b1062cbd... \
  --key YOUR_PRIVATE_KEY \
  --payload '{"v":1,"t":"whisper","d":"04ab...","a":{"from":"kaspatest:qq...","script":"...","spk":"...","deposit":20000000}}'

Option 2: Auto-fetch from block explorer

# decode.py automatically tries these sources in order:
# 1. --payload (manual)              ← fastest, fully offline
# 2. --info covenant_info.json       ← local file
# 3. API /api/whisper/{tx_id}        ← cache server
# 4. Block explorer (api-tn12)       ← on-chain fallback

python3 decode.py --tx b1062cbd... --key YOUR_KEY
# It just works™ — tries all sources automatically

How it reconstructs covenant_info from payload

# The `a` field contains everything:
# - script → covenant redeem script (for signing)
# - from   → sender address (refund destination)
# - spk    → sender SPK (for verification)
# - deposit → minimum refund amount
# b_pubkey is extracted from the covenant script itself
# p2sh_address is recomputed from the script

ECIES Encryption Details

# Encrypt: uses recipient's compressed public key (33 bytes)
from ecies import encrypt, decrypt

# Sender encrypts
compressed_pubkey = "02" + recipient_xonly_hex  # Add prefix to x-only
ciphertext = encrypt(compressed_pubkey, message.encode("utf-8"))

# Recipient decrypts (with their private key)
plaintext = decrypt(privkey_bytes, ciphertext)

⏰CLTV Timeout Reclaim (v3)

🔓 No More Stuck Deposits!

If the recipient doesn't read your message within the timeout, you can reclaim your 0.2 KAS deposit yourself. Fully trustless — no intermediary needed.

How It Works

The covenant script uses OP_IF / OP_ELSE with two spending paths:

✅ IF Branch — Bob Reads
  • Bob spends the UTXO
  • Covenant enforces refund → Alice
  • Available immediately
  • Sig script: <sig> OP_TRUE <redeem>
⏰ ELSE Branch — Alice Reclaims
  • Alice reclaims after timeout
  • CLTV checks DAA score ≥ timeout
  • TX lock_time must be set
  • Sig script: <sig> OP_FALSE <redeem>

Usage

# 1. Alice sends with CLTV timeout (auto-set to current_daa + 1000)
python3 covenant_send.py "Secret message with timeout!"
# → Saves covenant_info.json (keep this file!)

# 2. Bob reads normally (IF branch) — refund goes to Alice
python3 covenant_read.py
# → Needs covenant_info.json from Alice

# 3. If Bob doesn't read — Alice reclaims after timeout (ELSE branch)
python3 covenant_reclaim.py
# → Reads covenant_info.json, checks timeout, reclaims 0.2 tKAS

⚠️ Kaspa vs Bitcoin: CLTV Difference

Important Discovery

Kaspa's OP_CHECKLOCKTIMEVERIFY (0xb0) pops the stack value, unlike Bitcoin where it's a NOP-style opcode. This means OP_DROP is not needed after CLTV in Kaspa scripts.

📜Covenant Script Breakdown

The v3 redeem script uses OP_IF / OP_ELSE for dual spending paths:

IF Branch — Bob Reads (Refund to Alice)

OP_IF (0x63)Start IF branch

Part 1 — Verify Refund Goes to Sender

<A_spk_bytes>Push sender's ScriptPublicKey bytes (2B version + script)
OP_FALSE (0x00)Push output index 0
OP_TX_OUTPUT_SPK (0xc3)Introspect: get spending TX output[0]'s SPK
OP_EQUAL (0x87)Compare: does output[0] pay to sender?
OP_VERIFY (0x69)✅ Abort if refund goes to wrong address

Part 2 — Verify Refund Amount

OP_FALSE (0x00)Push output index 0
OP_TX_OUTPUT_AMOUNT (0xc2)Introspect: get output[0]'s amount
<20000000>Push 0.2 KAS (20,000,000 sompi)
OP_GTE (0xa2)Check: output amount ≥ 0.2 KAS?
OP_VERIFY (0x69)✅ Abort if refund too small

Part 3 — Verify Recipient Signature

<B_pubkey 32B>Push recipient's 32-byte Schnorr x-only pubkey
OP_CHECKSIG (0xac)✅ Only recipient can unlock this UTXO

ELSE Branch — Alice Reclaims (After Timeout)

OP_ELSE (0x67)Start ELSE branch
<timeout_daa>Push timeout DAA score (e.g. current + 1000)
OP_CHECKLOCKTIMEVERIFY (0xb0)Verify TX lock_time ≥ timeout (pops stack in Kaspa!)
<A_pubkey 32B>Push sender's Schnorr pubkey
OP_CHECKSIG (0xac)✅ Only sender can reclaim after timeout
OP_ENDIF (0x68)End conditional

Python Implementation

def build_covenant_script_with_timeout(a_spk_bytes, a_pubkey, b_pubkey, deposit, timeout_daa):
    s  = bytes([0x63])                  # OP_IF
    # IF: Bob reads → refund to Alice
    s += push_data(a_spk_bytes)           # Sender's SPK
    s += push_int(0)                      # Output index 0
    s += bytes([0xC3, 0x87, 0x69])      # OP_TX_OUTPUT_SPK, OP_EQUAL, OP_VERIFY
    s += push_int(0)                      # Output index 0
    s += bytes([0xC2])                  # OP_TX_OUTPUT_AMOUNT
    s += push_int(deposit)                # 20,000,000 sompi
    s += bytes([0xA2, 0x69])            # OP_GTE, OP_VERIFY
    s += push_data(b_pubkey)              # Recipient pubkey
    s += bytes([0xAC])                  # OP_CHECKSIG
    # ELSE: Alice reclaims after timeout
    s += bytes([0x67])                  # OP_ELSE
    s += push_int(timeout_daa)            # Timeout DAA score
    s += bytes([0xB0])                  # OP_CHECKLOCKTIMEVERIFY (pops in Kaspa!)
    s += push_data(a_pubkey)              # Sender pubkey
    s += bytes([0xAC])                  # OP_CHECKSIG
    s += bytes([0x68])                  # OP_ENDIF
    return s

⚙️TN12 Covenant Opcodes

Introspection opcodes that let scripts inspect the spending transaction:

OpcodeHexStack EffectDescription
OP_TX_OUTPUT_SPK0xc3<idx> → <spk>Get output's ScriptPublicKey
OP_TX_OUTPUT_AMOUNT0xc2<idx> → <amt>Get output's amount (sompi)
OP_TX_INPUT_COUNT0xb3→ <n>Number of inputs
OP_TX_OUTPUT_COUNT0xb4→ <n>Number of outputs
OP_TX_INPUT_AMOUNT0xbe<idx> → <amt>Get input's amount (sompi)

SPK Bytes Encoding

OP_TX_OUTPUT_SPK pushes version(2 bytes BE) + script_bytes:

// P2PK SPK bytes (36 bytes total):
0x0000                          // version = 0 (2 bytes big-endian)
0x20 <32-byte-pubkey> 0xac    // OP_DATA_32 + pubkey + OP_CHECKSIG
